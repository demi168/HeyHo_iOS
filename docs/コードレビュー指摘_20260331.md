# コードレビュー指摘（セキュリティ・メンテナンス性）

レビュー日: 2026-03-31  
観点: セキュリティ、メンテナンス性（Firestore rules / functions / iOS データアクセス / 認証・通知を重点確認）  
注: コード上で確認できる点のみ記載。

---

## 1

| 項目 | 内容 |
|------|------|
| **問題** | `onUserDeleted` が Firestore の **1バッチあたり最大500操作** を超えるデータ量だと `batch.commit()` が失敗しうる一方、処理が **複数バッチに分割されていない**。 |
| **影響** | Auth のユーザー削除は先に完了しうるため、友だち数・`heyhos` 件数が多い場合に **クリーンアップ失敗・部分成功** が起き、**Auth 上は削除済みだが Firestore にデータが残る**、または **途中までしか削除されない** などの不整合リスクがある（バッチ失敗時のリトライ・補償処理も `index.ts` 内にはない）。 |
| **根拠ファイル** | `functions/src/index.ts`（`batch1`〜`batch4` の各 `commit()`、`heyhos` は `sentSnap`/`receivedSnap` を1バッチにまとめている箇所） |
| **優先度** | **P0** |

## 2

| 項目 | 内容 |
|------|------|
| **問題** | **プレミアム（LetsGo）制限が UI のみ**で、`sendHeyHo` や Firestore ルールでは `messageType` / プレミアムを検証していない。 |
| **影響** | 正規クライアント以外（改変クライアントや SDK を直接利用）から、`friends` 関係が満たされれば **`letsGo` を含む任意の `messageType` で `heyhos` を作成可能**。アプリ内の課金ゲートを**サーバー側で強制できない**。 |
| **根拠ファイル** | `heyho_ios/Features/Friends/FriendsView.swift`（`sendLetsGo` 時の `storeService.isPremium` チェック）、`heyho_ios/Services/FirestoreService.swift`（`sendHeyHo` に課金条件なし）、`firestore.rules`（`heyhos` の `create` に `messageType` / 課金の制約なし） |
| **優先度** | **P1** |

## 3

| 項目 | 内容 |
|------|------|
| **問題** | `users/{userId}` に対し、**ログイン済みユーザーなら誰でも `get` 可能**（`list` のみ拒否）。 |
| **影響** | **ユーザーIDが分かる相手**の `displayName`・`iconColor`・`createdAt` などを、友だち関係なく読み取れる。最小公開にしたい場合はプライバシー上の過剰公開。 |
| **根拠ファイル** | `firestore.rules`（`match /users/{userId}` の `allow get: if isSignedIn();`） |
| **優先度** | **P1** |

## 7

| 項目 | 内容 |
|------|------|
| **問題** | Cloud Functions の `onHeyHoCreated` が **入力の妥当性検証なし**（例: `toUserId` / `fromUserId` の存在・友だち関係）で FCM 送信まで進む。 |

## 補足（コード上で確認できた良い点）

- **FCM トークン**は `users/.../private/data` に寄せ、ルールも `private` は本人のみ。根拠: `firestore.rules`、`heyho_ios/Services/FirestoreService.swift`、`functions/src/index.ts`（`private` から `fcmToken` 取得）。
- **iOS の Firestore アクセス**は `FirestoreService` に集約され、パス参照が散らばっていない。根拠: `heyho_ios/Services/FirestoreService.swift`。
- **`heyhos` の更新・削除はルールで拒否**、**招待コードの `list` 拒否**など、ルール側の基本方針はコードに明示されている。根拠: `firestore.rules`。
- **`GoogleService-Info.plist` は `.gitignore` に含まれる**。根拠: `.gitignore`。
