## 3. プロジェクト構成

### ディレクトリ構造

```
heyho_ios/
├── App/          # エントリポイント、ルート表示
├── Auth/         # 認証（SignIn, AuthState, Apple Sign In）
├── Models/       # データモデル（Firestore 用）
├── Services/     # シングルトンサービス
├── Features/     # 機能ごとの UI
│   └── {Feature}/
│       └── XxxView.swift
└── Utilities/    # ViewModifiers など共通ユーティリティ
```

### 命名規則

| 種類 | ルール | 例 |
|------|--------|-----|
| View | `XxxView.swift` | FriendsView, ProfileView |
| 行コンポーネント | `XxxRow` | FriendRow, InboxRow |
| Service | `XxxService`、シングルトン `.shared` | FirestoreService.shared |
| Model | `AppXxx` またはドメイン名 | AppUser, HeyHo |
| 状態 enum | `XxxState` | FriendRowState |
| Firestore コレクション | 複数形・camelCase | users, heyhos, inviteCodes |

## 4. Swift / SwiftUI の規約

### 状態管理

- **AuthState**: `@EnvironmentObject` で注入。全画面で共通
- **画面ローカル**: `@State` で保持（例: `friends`, `errorMessage`, `isLoading`）
- **エラー表示**: `@State private var errorMessage: String?` + `.errorAlert($errorMessage)` を利用

### 非同期処理

- Firestore などの非同期は `async/await` で記述
- View から呼ぶときは `Task { }` でラップ
- UI 更新は `await MainActor.run { }` で明示（必要に応じて）
- `defer { }` で `isLoading = false` などを確実に実行

### 空状態・ローディング

- ローディング中: `ProgressView().frame(maxWidth: .infinity, maxHeight: .infinity)`
- 空リスト: `ContentUnavailableView("メッセージ", systemImage: "xxx", description: Text("..."))`
- 一覧は `.refreshable { await loadXxx() }` でプルリフレッシュ対応

### 開発用コード

- DEBUG 専用は `#if DEBUG ... #endif` で囲む
- ダミーデータは `private let` で画面内に定義する例を採用（必要なら ViewModel へ移行）

## 5. Firestore / Firebase

### モデル

- `Codable` 準拠、`@DocumentID var id: String?` で Firestore ドキュメント ID をマッピング
- `CodingKeys` でフィールド名を明示（camelCase ↔ snake_case の変換用）
- `createdAt` などは `Date` 型。サーバータイムスタンプの場合は初回読み取り時に `nil` になり得る

### FirestoreService

- 全 Firestore 操作は `FirestoreService.shared` 経由
- メソッドは `async throws`、エラーは呼び出し元でハンドリング
- コレクション名は定数化せず、メソッド内で直接指定（現状のスタイルに合わせる）

### セキュリティルール

- **新コレクション追加時は `firestore.rules` を必ず更新する**
- ルール追加後は `firebase deploy --only firestore:rules` でデプロイ
- read 条件: `toUserId == auth.uid || fromUserId == auth.uid` など、双方が関与するデータは両方で許可することを検討

### Cloud Functions

- `functions/src/index.ts` にトリガーを追加
- Firestore トリガーは `onCreate` / `onUpdate` などで適切に設定
- メッセージ種別（hey / ho / letsGo）ごとに通知文言を分岐

## 6. UI パターン

### ナビゲーション

- メインは `NavigationStack`
- モーダルは `.sheet(isPresented:)`、遷移は `NavigationLink`
- ツールバーは `.toolbar { ToolbarItem(placement: .topBarTrailing) { ... } }`

### フォーム・入力

- `TextField` は `.textFieldStyle(.roundedBorder)` を基本
- ボタンは `.buttonStyle(.borderedProminent)` または `.buttonStyle(.bordered)` で統一
- 送信ボタンは `disabled(条件)` で二重送信防止

### エラー表示

- `ViewModifiers` の `.errorAlert($errorMessage)` を使う
- エラー時は `errorMessage = error.localizedDescription` または日本語に変換したメッセージをセット

## 7. ビルド・デプロイ

- Xcode: `HeyHo` スキームでビルド
- Functions: `cd functions && npm run build && npm run deploy`
- Firestore ルール: `npx firebase-tools deploy --only firestore:rules`
